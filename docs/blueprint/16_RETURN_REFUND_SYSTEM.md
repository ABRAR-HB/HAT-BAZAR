# HAT-BAZAR Blueprint

# Return & Refund System

Version: 1.0

Status: Final Blueprint

---

# Purpose

The Return & Refund System protects both Customers and Sellers by providing a transparent, fair and secure return process.

The system ensures that every Return and Refund follows standardized marketplace rules.

---

# Scope

This Blueprint defines:

- Return Eligibility
- Return Request
- Return Approval
- Return Pickup
- Refund Rules
- Dispute Resolution
- Administrative Review
- Business Rules

---

# Core Principles

The Return & Refund System follows these principles:

- Fairness
- Transparency
- Customer Protection
- Seller Accountability
- Marketplace Integrity

Every Return Request must be supported by sufficient evidence.

---

# Return Eligibility

A Return Request may be submitted only when:

- The Order has been successfully delivered.
- The Product is eligible under the HAT-BAZAR Return Policy.
- The Return Request is submitted within the allowed Return Period.

Products outside the Return Policy cannot be returned.

---

# Return Period

Customers may submit a Return Request within:

- 3 days after successful Delivery.

After 3 days:

- Return Requests are no longer accepted.

---

# Return Request Requirements

Every Return Request must include:

- Return Reason (Mandatory)
- At least one Product Photo (Mandatory)
- Written Explanation (Mandatory)

Incomplete Return Requests cannot be submitted.

---

# Return Reasons

Supported Return Reasons include:

- Wrong Product Delivered
- Damaged Product
- Defective Product
- Missing Items
- Product Not as Described
- Wrong Size or Variant
- Other (Explanation Required)

---

# General Return Rules

- Only verified Customers may request Returns.
- Every completed Order may have only one Return Request.
- Return Requests are permanently recorded.
- Every Return Request receives a unique Return ID.

---

**End of Part 1**

# Return Approval

Every Return Request is reviewed by an authorized Admin.

The Admin evaluates:

- Return Reason
- Product Photos
- Written Explanation
- Order History
- Delivery Records

The Admin may:

- Approve the Return Request.
- Reject the Return Request.
- Request additional information if necessary.

Every decision is permanently recorded.

---

# Return Status

Every Return Request follows the official Return Workflow.

Return Status includes:

- Return Requested
- Under Admin Review
- Return Approved
- Return Rejected
- Rider Assigned
- Pickup Scheduled
- Pickup Completed
- Returning to Seller
- Return Delivered
- Refund Processing
- Refund Completed
- Return Closed

Status changes are automatically recorded.

---

# Return Pickup

All Return Pickups are performed only by HAT-BAZAR Riders.

Seller Delivery Services are not permitted.

Customers cannot personally deliver Return Products to Sellers.

The HAT-BAZAR Delivery Network manages the complete Return Logistics.

---

# Rider Workflow

After Return Approval:

- The system assigns a HAT-BAZAR Rider.
- The Rider schedules Product Pickup.
- The Rider collects the Product.
- The Rider delivers the Product back to the Seller.

Every Rider action is permanently recorded.

---

# Return Tracking

Customers and Sellers may track the Return process.

Tracking Information includes:

- Current Status
- Assigned Rider
- Pickup Time
- Delivery Time
- Return Timeline

Tracking updates are provided automatically.

---

# Return Delivery Charge

Return Delivery Charges are fully covered by HAT-BAZAR.

Neither the Customer nor the Seller pays an additional Return Delivery Fee.

---

# General Workflow Rules

- Every Return Request follows the official Return Workflow.
- Every status change is permanently stored.
- Manual workflow manipulation is prohibited.
- Return history cannot be deleted.

---

**End of Part 2**


# Refund Rules

Refund is initiated only after:

- The Return Product is successfully delivered back to the Seller.
- The Return process is officially completed.

Refunds are never issued before the Return process is completed.

---

# Refund Method

Customers do not have a Wallet.

Refunds are always sent to the same Payment Account used for the original Order.

Supported Refund Methods include:

- bKash
- Nagad
- Bank Card
- Bank Account
- Other supported payment methods

Changing the Refund destination is not allowed.

---

# Seller Responsibility

If the Admin determines that the Return occurred due to the Seller's fault, the Seller becomes responsible for the incident.

Examples include:

- Wrong Product Delivered
- Damaged Product
- Defective Product
- Missing Items
- Product Not as Described

The incident is permanently recorded in the Seller Performance History.

---

# Seller Performance Penalty

If a Seller is repeatedly responsible for valid Returns:

- The Seller's Badge Level may be reduced.
- Additional administrative actions may be taken.

Badge reduction is performed only after an Admin decision.

Automatic Badge reduction is not allowed.

---

# Dispute Resolution

If a dispute occurs between the Customer and the Seller:

- The case is forwarded to Admin Review.

The Admin examines:

- Return Reason
- Product Photos
- Written Explanation
- Order History
- Delivery Records
- Other available evidence

The Admin's decision is final.

---

# Administrative Audit

Every Return and Refund action is permanently recorded.

Audit information includes:

- Return ID
- Refund ID
- Admin ID
- Action Taken
- Reason
- Date & Time

Audit records cannot be edited or deleted.

---

# Financial Rules

- Customer payments remain in HAT-BAZAR Escrow until settlement.
- If a Return is approved, the Product value is refunded to the Customer.
- Refunds are sent only to the original Payment Account.
- Return Delivery Charges are paid by HAT-BAZAR.
- Delivery Charges follow the marketplace Refund Policy.

---

# Business Rules

- Refunds require successful Return completion.
- Admin has final authority over Return and Refund disputes.
- Seller accountability is permanently recorded.
- Refund history is permanently stored.
- Financial records cannot be deleted.

---

**End of Part 3**


# Return Integrity

The Return & Refund System must protect both Customers and Sellers through fair, transparent and standardized procedures.

Fraudulent Return Requests and fraudulent Seller activities are strictly prohibited.

---

# Marketplace Integrity

The Return process must always remain neutral.

Neither the Customer nor the Seller receives unfair advantages.

Every Return is evaluated based on available evidence and marketplace policies.

---

# AI Limitations

AI may assist in:

- Detecting suspicious Return patterns
- Identifying repeated fraudulent activities
- Supporting Admin investigations

AI cannot:

- Approve Returns
- Reject Returns
- Approve Refunds
- Reject Refunds
- Reduce Seller Badge Levels

Final authority always belongs to Admin.

# Final Business Rules

- Customers may request Returns only within 3 days after successful Delivery.
- Return Requests require a Reason, at least one Product Photo and a Written Explanation.
- Only Products covered by the HAT-BAZAR Return Policy are eligible for Return.
- Every Return Request is reviewed by an Admin.
- Return Pickup is performed only by HAT-BAZAR Riders.
- Return Delivery Charges are paid by HAT-BAZAR.
- Refunds are issued only after successful Return completion.
