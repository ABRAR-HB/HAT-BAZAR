# NOTIFICATIONS TABLE

Version: 1.0

Status: Official Database Design

---

# Purpose

The notifications table stores every notification generated by the HAT-BAZAR platform.

Notifications keep users informed about important events such as orders, payments, chat messages, promotions, wallet activities, refunds, and administrative announcements.

The system is designed to support both real-time and scheduled notifications.

---

# Table Name

notifications

---

# Primary Key

id

BIGSERIAL

Auto Increment

---

# Description

Each notification belongs to one recipient.

Notifications may be generated automatically by the system or manually by administrators.

Notifications remain available for historical records even after being read.

---

# Ownership

Each notification belongs to:

One recipient

Optional sender

Optional related entity

# Business Rules

Every notification has one recipient.

System notifications may not have a sender.

Notifications must support soft delete.

Read status must be tracked.

Notifications should remain available until deleted by the user or archived by policy.

# Constraints

Every notification must have one recipient.

Title and message cannot be empty.

Notification UUID must be unique.

Only supported notification types are allowed.

Read time can only exist after the notification is marked as read.

Deleted notifications remain available for auditing.

---

# Security & Privacy

Users can access only their own notifications.

Authentication is required for every notification API.

Notification data must never expose another user's private information.

Soft deleted notifications remain hidden from users while remaining available for administrative audit.
