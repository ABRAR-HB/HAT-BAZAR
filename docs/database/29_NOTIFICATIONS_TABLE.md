# NOTIFICATIONS TABLE

Version: 1.0

Status: Official Database Design

---

# Purpose

The notifications table manages all notifications sent through the HAT-BAZAR platform.

It supports in-app notifications, push notifications, email notifications, SMS notifications, and future communication channels while maintaining delivery status, read status, and complete notification history.

---

# Table Name

notifications

---

# Primary Key

id

BIGSERIAL

Auto Increment

---

# Description

Each notification record represents one message delivered to one recipient.

A notification may be generated by user actions, system events, orders, payments, deliveries, promotions, or automated background services.

Every notification maintains its complete delivery history.

---

# Ownership

Each notification belongs to:

One recipient

Optional sender

Optional related entity

Optional notification template

Notifications should be immutable after delivery.

Read status should be updated independently.

Delivery status must always be tracked.

Soft delete is not recommended.

---

# Constraints

Notification UUID must be unique.

Every notification must have one recipient.

Title and message cannot be empty.

Read time cannot be earlier than sent time.

Delivered notifications cannot return to pending status.

---

# Notification Delivery Rules

Notifications are generated by business events.

Each notification is queued before delivery.

The platform records every delivery attempt.

Delivery status is updated after each attempt.

Read status is updated independently from delivery status.

---

# Retry Mechanism

If notification delivery fails:

Retry automatically according to platform policy.

Record each retry attempt.

Stop retrying after the maximum retry limit.

Escalate persistent failures for administrative review.

---

# Security & Privacy Requirements

Users may access only their own notifications.

Sensitive notification content should be transmitted securely.

Notification history should remain available for auditing.

Administrative access must follow role-based permissions.

Personal information included in notifications should comply with privacy regulations.
